# Plan: Converter improvements and `--compat` flag

Date: 2025-09-09
Author: automated coding agent

Summary
-------
This plan documents the changes made to `splurge-unittest-to-pytest` to address issues discovered when converting `unittest.TestCase`-based tests to pytest (see `docs/issue-u2p.md`). The goal is to make the converter safer and provide an opt-in compatibility mode to preserve `self.<attr>` style access without rewriting all test methods.

Goals
-----
- Skip binary and `__pycache__` files during discovery to avoid UnicodeDecodeError.
- Ensure `import pytest` is inserted before any generated `@pytest.fixture()` decorators and respect module docstrings.
- Provide a `--compat` CLI flag to emit an autouse fixture that attaches fixture values to `request.instance`, preserving `self.<attr>` semantics.
- Add unit tests to exercise these scenarios and prevent regressions.

Design
------
1. Discovery:
   - Update `find_unittest_files` to ignore paths that contain `__pycache__` and to treat unreadable files (PermissionError/UnicodeDecodeError) as non-matching rather than raising.

2. Import ordering and fixtures:
   - When the transformer determines `pytest` is required, insert `import pytest` at module top-level after any module docstring and before any generated fixtures.
   - Create fixtures from `setUp` assignments; when `teardown` cleanup references attributes used by fixtures, create yield-style fixtures.

3. Compatibility autouse fixture (`--compat`):
   - Add CLI option `--compat/--no-compat` (default: on) to control emission of a small autouse fixture called `_attach_to_instance`.
   - The autouse fixture will, for each fixture generated from `setUp`, call `setattr(request.instance, '<name>', <fixture>)` when `request.instance` exists.
   - This keeps converted tests that still use `self.attr` working without rewriting all methods.

4. Tests:
   - Add unit tests testing file discovery ignoring `__pycache__` and unreadable files.
   - Add tests for fixtures with cleanup (yield pattern), multiple `setUp` assignments producing multiple fixtures, and simple variable name regression scans (e.g., ensure generated tests use consistent names present in the module).

Implementation notes
--------------------
- Transformer now accepts a `compat: bool` parameter. When False, do not emit `_attach_to_instance`.
- Keep changes limited to `main.py`, `converter.py`, and `cli.py` to reduce risk.
- Add tests under `tests/unit/` to cover new behavior.

Acceptance criteria
-------------------
- All unit tests pass (`pytest` green).
- No regressions in conversion output for the existing test corpus.
- New tests for discovered edge cases are present and pass.

Follow-ups
----------
- Consider adding a `--compat=autouse|explicit` mode where compatibility is opt-in per-class instead of global autouse.
- Add a converter-level option to emit `self`-to-fixture bindings directly in class bodies instead of an autouse fixture for cleaner, explicit conversions.
- Add CLI logging when files are skipped due to unreadable content to aid users during bulk conversions.
